# 智见 — 仓储质检 AI 系统 Web Demo

## 系统概述

智见是一套面向汽车制造企业仓储场景的 AI 质检软件系统 Web 演示，覆盖从原料入厂到成品出库的全环节质量管控。本系统模拟三端设备（PDA手持终端、Station检测工位、Admin管理后台）的完整交互体验。

**部署地址**: https://f5rots4guzqoo.ok.kimi.link

---

## 技术栈

- **框架**: React 19 + TypeScript
- **构建**: Vite 7
- **样式**: Tailwind CSS v3.4.19
- **UI组件**: shadcn/ui
- **图表**: Recharts
- **动画**: Framer Motion
- **路由**: React Router (HashRouter)
- **图标**: Lucide React
- **数据**: 纯前端 Mock（Context + localStorage）

## PDA端功能

模拟工业手持设备，25+个路由页面：

- **首页** (`/pda`): 2×2业务入口（收货、上架、拣货、巡检）
- **收货链路**: 扫码收货 → 收货结果摘要 → 签收汇总 → 到货预约登记
- **上架链路**: 上架任务列表 → 扫码上架
- **拣货链路**: 拣货任务列表 → 拣货执行（小型路线图）
- **巡检链路**: 巡检任务列表 → 巡检执行 → 盘点复点
- **异常处理**: 异常列表 → 异常结果 → 异常上报编辑 → 问题件移交
- **防伪溯源**: 扫码 → 验真（新旧二维码版本区分）
- **盘点**: 盘点复点 + PATCH页面（盘点任务下发/结果处理/标签补打）
- **生产退料**: PATCH页面（退料扫码/退料结果摘要）
- **我的**: 个人信息、设备状态、工作摘要

硬件参数卡展示：型号、防护等级≥IP65、续航≥8h、摄像头、WiFi+4G/5G

## Station端功能

模拟桌面检测工位，4种模式：

1. **待机页** (`/station`): 品牌区、今日数据、设备状态、Agent摘要
2. **收货检测** (`/station/receive`): 三栏布局 + 7种检测状态手动切换
3. **出库复核** (`/station/outbound`): 四环节时间线 + 出库单信息
4. **包装复核/异常分流** (`/station/triage`): IQ-*/CL-*编号 + 分流处理

**7种检测状态**（可手动切换）：
- 通过态（绿色）/ 警示态（橙色）/ 不清晰态（黄色+OCR进度条）
- 标签缺失态（橙红全屏弹窗+合规条款）
- 外观缺陷态（橙色全屏+NG件标注+缺陷框选）
- 拦截态（红色全屏弹窗+蜂鸣+L1强拦截徽章）
- 多模态异常态（红色全屏+消防联动+L1徽章）

硬件参数卡展示：固定相机≥500万像素+ROI配置、桌面检测台≤600×400mm+补光灯可调

## Admin端功能

管理后台，13个路由页面：

1. **首页工作台** (`/admin`): Agent提醒、今日拦截价值(¥23.6万)、待确认建议、快捷入口
2. **Agent协作台** (`/admin/agents`): 事件列表+详情+处理区
3. **工单中心** (`/admin/workorders`): WO-001~005工单管理
4. **数据看板** (`/admin/dashboard`): KPI指标 + AI vs人工对照表
5. **规则配置** (`/admin/rules`): 13条If-Then规则编辑
6. **供应商分析** (`/admin/suppliers`): 异常率排行 + 建议动作卡
7. **异常案例库** (`/admin/gallery`): 缩略图卡片归档
8. **质量标准库** (`/admin/standards`): 品类审核标准查询
9. **问题件/隔离区** (`/admin/quarantine`): IQ-*滞留时长管理
10. **索赔协同** (`/admin/claims`): 照片+监控+责任方+沟通记录
11. **盘点策略** (`/admin/recount`): 动碰/静态/重点库位配置
12. **视频违规分析** (`/admin/video`): 模拟视频流+违规时间线+截图证据
13. **采购计划对接** (`/admin/spf`): 拦截数据+偏差分析+手动同步

## 启动方式

```bash
# 安装依赖
npm install

# 开发模式
npm run dev

# 构建
npm run build

# 预览构建结果
npm run preview
```

构建产物位于 `dist/` 目录，可通过任何静态服务器部署。
